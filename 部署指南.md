## 1. 创建 Supabase 项目

### 步骤 1: 注册/登录 Supabase
1. 访问 https://supabase.com
2. 点击右上角 **"Start your project"**
3. 使用 GitHub 或 Email 注册/登录

### 步骤 2: 创建新项目
1. 登录后，点击 **"New Project"**
2. 填写项目信息：
   - **Name**: `cube-competition-tracker`（可自定义）
   - **Database Password**: 设置一个强密码（**请保存好！**）
   - **Region**: 选择 `Northeast Asia (Seoul)` 或 `Singapore`（离国内较近）
   - **Pricing Plan**: 选择 **Free**（免费套餐）
3. 点击 **"Create new project"**
4. 等待项目创建完成（约 1-2 分钟）

## 2. 初始化数据库

### 步骤 3: 获取数据库连接信息
1. 在项目控制台左侧菜单，点击 **"Settings"** → **"API"**
2. 复制以下信息（稍后需要用到）：
   - **Project URL**: `https://xxxxx.supabase.co`
   - **anon public key**: `eyJhbGciOiJIUzI1NiIs...`

### 步骤 4: 运行 SQL 初始化脚本
1. 在项目控制台左侧菜单，点击 **"SQL Editor"**
2. 点击 **"New query"**
3. 打开本项目的 `schema.sql` 文件（已提供）
4. 将文件内容**完整复制**到 SQL Editor 中
5. 点击 **"Run"** 执行
6. 看到 `数据库 schema 创建成功！` 提示即表示成功

### 步骤 5: 验证表创建成功
1. 在左侧菜单点击 **"Table Editor"**
2. 应该看到 3 张表：
   - `competitions`（比赛信息）
   - `participants`（选手信息）
   - `attempts`（成绩记录）
3. 还有 2 个视图：
   - `competition_stats`（比赛统计）
   - `participant_stats`（选手统计）

## 3. 部署网页

### 方法一：本地直接使用（最简单）

1. 打开 `index.html` 文件
2. 在页面顶部的 **"Supabase 配置"** 区域填写：
   - **Supabase URL**: 步骤 3 中复制的 URL
   - **Anon Key**: 步骤 3 中复制的 anon key
3. 点击 **"连接数据库"** 按钮
4. 连接成功后即可开始使用！

> ✅ 优点：无需部署，双击打开即用  
> ⚠️ 缺点：每次打开网页都需要重新连接

### 方法二：部署到 GitHub Pages（推荐 - 免费永久托管）

#### 步骤 6: 创建 GitHub 仓库
1. 注册/登录 https://github.com
2. 点击右上角 **"+"** → **"New repository"**
3. 填写仓库名：`cube-competition-tracker`
4. 选择 **Public**
5. 点击 **"Create repository"**

#### 步骤 7: 上传文件
1. 在仓库页面，点击 **"uploading an existing file"**
2. 上传以下文件：
   - `index.html`（已提供）
3. 点击 **"Commit changes"**

#### 步骤 8: 启用 GitHub Pages
1. 在仓库页面，点击 **"Settings"**
2. 左侧菜单找到 **"Pages"**
3. **Source** 选择 **"Deploy from a branch"**
4. **Branch** 选择 **"main"** 和 **"/ (root)"**
5. 点击 **"Save"**
6. 等待约 1-2 分钟，页面会显示：
   ```
   Your site is live at https://你的用户名.github.io/cube-competition-tracker/
   ```

#### 步骤 9: 配置 API 密钥（重要！）
由于 GitHub Pages 是公开访问的，你需要：

1. 打开部署后的网页
2. 在 **"Supabase 配置"** 区域填写 URL 和 Key
3. 点击 **"连接数据库"**
4. 浏览器会自动保存配置到本地（LocalStorage）
5. 下次打开网页会自动填充

> 🔒 **安全提示**：anon key 是公开安全的，但建议在生产环境中设置更严格的 RLS 策略。

### 方法三：部署到 Vercel（最快）

1. 注册/登录 https://vercel.com（使用 GitHub 登录）
2. 点击 **"New Project"**
3. 导入你的 GitHub 仓库
4. 点击 **"Deploy"**
5. 约 30 秒后部署完成，获得 `https://xxx.vercel.app` 链接

## 4. 使用说明

### 添加比赛
1. 点击 **"添加比赛"** 标签
2. 填写比赛信息：
   - **比赛编号**: 唯一标识，如 `COMP-2026-001`
   - **比赛名称**: 如 `2026春季魔方赛`
   - **比赛日期**: 选择日期
   - **比赛地点**: 可选
   - **主办方**: 可选
3. 点击 **"添加比赛"**

### 添加成绩
1. 点击 **"添加成绩"** 标签
2. 选择对应的比赛
3. 填写选手信息：
   - **选手名称**: 必填
   - **尝试序号**: 第几次尝试（1, 2, 3...）
   - **魔方类型**: 选择 `智能魔方` 或 `非智能魔方`
4. 如果选择 **智能魔方**，会显示额外字段：
   - **步数**: 必填
   - **TPS**: 必填
   - **复原步骤详情**: 可选（JSON 格式）
5. 如果选择 **非智能魔方**，可以填写：
   - **录像链接**: 可选（用于复盘）
6. 填写打乱公式和复原时间
7. 选择是否有惩罚（+2 或 DNF）
8. 点击 **"添加成绩"**

### 查看数据
1. 点击 **"查看数据"** 标签
2. 点击以下按钮之一：
   - **加载比赛列表**: 查看所有比赛
   - **加载成绩记录**: 查看所有成绩（最新 100 条）
   - **加载统计**: 查看统计数据

## 5. 常见问题

### Q1: 连接数据库失败怎么办？
**A**: 检查以下几点：
- Supabase URL 是否完整（包括 `https://`）
- Anon Key 是否完整复制（通常以 `eyJ` 开头）
- 网络连接是否正常（Supabase 可能被墙，需要科学上网）

### Q2: 添加比赛时提示"编号已存在"？
**A**: 比赛编号（`competition_number`）必须唯一。请使用不同的编号，如 `COMP-2026-002`。

### Q3: 添加成绩时提示"尝试编号已存在"？
**A**: 同一比赛中的尝试序号不能重复。检查是否重复添加了相同的尝试序号。

### Q4: 智能魔方必须填写步数和 TPS 吗？
**A**: 是的！根据数据库设计，智能魔方必须有步数和 TPS 数据。如果确实没有，可以填写估算值，或者改用"非智能魔方"类型。

### Q5: 如何备份数据？
**A**: 在 Supabase 控制台：
1. 点击 **"Settings"** → **"Database"**
2. 找到 **"Database backups"** 部分
3. Free 计划保留最近 24 小时的备份

或者手动导出：
1. 点击 **"Table Editor"**
2. 选择表 → 点击右上角 **"..."** → **"Export table"**

### Q6: 网页能离线使用吗？
**A**: 不能。网页需要连接 Supabase 数据库才能工作。但配置信息会保存在浏览器本地，下次打开自动填充。

### Q7: 如何共享给其他人使用？
**A**: 将部署后的网页链接发给其他人即可。他们只需要：
1. 打开网页
2. 填写相同的 Supabase URL 和 Anon Key
3. 开始使用（所有数据存在同一个数据库中）
