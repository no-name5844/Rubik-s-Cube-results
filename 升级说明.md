# 项目自定义配置 + 尝试编号改进说明

## 📋 改进概述

### 1. 项目高度可自定义
现在每个项目（events）都可以通过 JSON 配置自定义参数，包括：
- 是否支持智能魔方
- 是否记录步数/TPS
- 支持的惩罚类型
- 尝试编号的标签和类型
- 最大尝试次数
- 自定义字段（高级用法）

### 2. 尝试编号改进
- 从"第几次尝试"（数字）改为"尝试编号"（文本）
- 支持灵活的编号方式：1, 2, A, B, Round1-1, 等
- 数据库字段类型：INTEGER → TEXT

---

## 🗄️ 数据库变更

### 新增升级脚本 `schema-v4-upgrade.sql`
用于升级**已有数据库**，包括：
1. 为 `events` 表添加 `event_config` JSONB 字段
2. 将 `attempts.attempt_number` 从 INTEGER 改为 TEXT
3. 创建 `event_config_view` 视图（方便查询配置）
4. 创建 `get_event_form_config()` 函数（动态生成表单）

### 更新 `schema-v3.sql`（完整版）
- 新安装时直接使用最新结构
- `events` 表包含 `event_config` 字段
- `attempts` 表使用 TEXT 类型的 `attempt_number`

## 🎯 使用方法

### 一、升级已有数据库

如果你的 Supabase 数据库已经创建（使用旧的 `schema-v3.sql`），需要执行升级脚本：

1. 打开 Supabase Dashboard → SQL Editor
2. 复制 `schema-v4-upgrade.sql` 的内容
3. 粘贴到 SQL Editor
4. 点击 "Run" 执行

**升级脚本会：**
- 自动为现有项目添加默认配置
- 安全转换 `attempt_number` 字段类型（原有数字会自动转为文本）
- 刷新 Supabase schema 缓存

### 二、配置项目参数

1. 打开网页 `index-v3.html`
2. 连接到数据库
3. 点击 **"🎯 项目管理"** 标签页
4. 在右侧找到 **"⚙️ 配置项目参数"** 区域
5. 选择要配置的项目
6. 编辑 JSON 配置（或点击 "📋 加载默认配置"）
7. 点击 **"💾 保存配置"**

#### 配置示例（JSON 格式）

```json
{
  "supports_smart_cube": true,
  "record_steps": true,
  "record_tps": true,
  "record_video": false,
  "penalty_types": ["none", "+2", "DNF"],
  "attempt_id_type": "number",
  "attempt_id_label": "尝试编号",
  "max_attempts": 5,
  "best_of": 1,
  "custom_fields": []
}
```

#### 配置项说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `supports_smart_cube` | boolean | 是否支持智能魔方（显示步数/TPS 字段）|
| `record_steps` | boolean | 是否记录复原步数 |
| `record_tps` | boolean | 是否记录 TPS（步/秒）|
| `record_video` | boolean | 是否记录视频链接（非智能魔方）|
| `penalty_types` | array | 支持的惩罚类型，如 `["none", "+2", "DNF"]` |
| `attempt_id_type` | string | 尝试编号类型：`"number"`(数字) / `"text"`(文本) / `"custom"`(自定义) |
| `attempt_id_label` | string | 尝试编号的显示标签，如 `"尝试编号"`, `"轮次"`, `"Round"` |
| `max_attempts` | number | 最大尝试次数（可选）|
| `best_of` | number | 取最佳 N 次（如：三局两胜）|
| `custom_fields` | array | 自定义字段列表（高级用法，暂未实现）|

### 三、录入成绩时使用尝试编号

1. 点击 **"📝 成绩录入"** 标签页
2. 选择比赛、项目、选手
3. 在 **"尝试编号"** 字段输入编号：
   - 数字：`1`, `2`, `3`, `4`, `5`
   - 文本：`A`, `B`, `Round1`, `Final`
   - 自定义：`2026-001`, `GroupA-1`
4. 填写其他成绩信息
5. 点击 **"➕ 添加成绩"**

## 🚨 注意事项

### 1. 升级现有数据库前请备份
```sql
-- 备份 events 表
CREATE TABLE events_backup AS SELECT * FROM events;

-- 备份 attempts 表
CREATE TABLE attempts_backup AS SELECT * FROM attempts;
```

### 2. 尝试编号改为文本后的影响
- **好处**：更灵活，支持各种编号方式
- **注意**：如果需要按数字排序，查询结果可能需要 `ORDER BY attempt_number::INTEGER`（仅当编号是纯数字时）

### 3. 项目配置的高级用法
- `event_config` 是 JSONB 字段，可以存储任意 JSON
- 未来可以扩展前端，根据配置动态显示/隐藏表单字段
- 当前版本需要手动编辑 JSON，未来可以添加可视化配置界面
